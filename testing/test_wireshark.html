Wireshark-lab for TEK – HTTP og TCP

Tre praksisøvelser (ENG UI) med bilder fra nettet. Fasit nederst på siden.

Denne siden er laget for Teknologiforståelse (TEK). Elevene skal bruke Wireshark til å analysere HTTP-over-TCP. Bildene er hentet fra åpne kilder på nett (engelsk grensesnitt) og viser hva elevene skal se etter.

Øvelse 1 – Grunnleggende HTTP GET (enkel)

Mål: Finne og tolke en HTTP GET-forespørsel og sentrale headere i Wireshark.

Instruksjoner

  1. Start Wireshark, velg riktig nettverksgrensesnitt.
  2. Sett display-filter: http.
  3. Åpne en nettleser og gå til http://example.com (HTTP, ikke HTTPS).
  4. Finn en GET-pakke. Utvid Hypertext Transfer Protocol i Packet Details.
  5. Noter: (a) klient- og server-IP, (b) metode og URI, (c) Host, (d) User-Agent.

Les mer: Praktiske tips for å filtrere og lese HTTP-headere i Wireshark hos NetworkProGuide (How to Filter HTTP Traffic in Wireshark).

Øvelse 2 – HTTP-headere og TCP-detaljer (medium)

Mål: Se hvordan HTTP ligger oppå TCP og identifisere TCP-felter.

Instruksjoner

  1. Bruk samme opptak som i øvelse 1.
  2. Klikk en HTTP-pakke og utvid Transmission Control Protocol.
  3. Noter: (a) kilde- og destinasjonsport, (b) flagg (ACK/PSH osv.), (c) Sequence/Ack-numre, (d) Window size.

Les mer: Trinn-for-trinn om TCP-analyse i Wireshark (treveis håndtrykk, flagg, osv.) hos GeeksforGeeks (TCP Analysis using Wireshark).

Øvelse 3 – Follow → TCP Stream (utfordrende)

Mål: Følg en HTTP-samtale (request + response) slik applikasjonslaget ser den.

Instruksjoner

  1. Generér trafikk til en ren HTTP-side (f.eks. http://httpforever.com).
  2. Høyreklikk en HTTP-pakke → Follow → TCP Stream.
  3. Identifiser i dialogen: full forespørsel (headere), statuslinje/kode, Content-Length/Content-Type, og språk fra Accept-Language.
  4. Skriv 3–4 setninger som forklarer hvordan TCP (pålitelige segmenter) og HTTP (applikasjonsprotokoll) samarbeider.

Les mer: Offisiell brukerveiledning om Follow Stream (inkl. HTTP/2) hos Wireshark (Following Protocol Streams).

Vurderingskriterier (rubrics)

Øvelse 1 (5 p): IP-adresser (2p), metode/ressurs (2p), Host (1p).

Øvelse 2 (6 p): TCP-porter (2p), flagg (2p), User-Agent (1p), Accept-Encoding (1p).

Øvelse 3 (10 p): Full forespørsel (3p), responskode/status (2p), Content-Length (2p), språkpreferanse (1p), forklaring TCP/HTTP (2p).

Total: 21 poeng.

Fasit

Klikk for å vise/skjule fasit
  • Øvelse 1: Klient = egen maskin; Server = example.com; Metode: GET; Ressurs: /; Host: example.com; User‑Agent: nettleserstreng.
  • Øvelse 2: TCP‑porter: src = ephemeral (tilfeldig), dst = 80; Flagg: ofte ACK/PSH når data sendes; Seq/Ack øker med byte; Window = mottakerbuffer.
  • Øvelse 3: Forespørsel starter med GET / HTTP/1.1 + Host, User‑Agent, Accept*; Respons typisk HTTP/1.1 200 OK + Content‑Length/Type; Språk fra Accept‑Language; Forklaring: HTTP (appl.) over TCP (pålitelige segmenter, rekkefølge og leveringssikring).
