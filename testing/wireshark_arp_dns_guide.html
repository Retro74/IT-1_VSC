Wireshark-guide: Oppdage ARP-spoofing og DNS-tunneling

Instruksjoner, filtre og tolkning med skjermbilder fra Wireshark.

1. ARP-spoofing

Capture filter

arp

Display filter

arp.opcode == 2

Hva se etter? Mange ARP-replies uten tilsvarende requests, eller samme IP med ulike MAC-adresser.

2. DNS-tunneling

Capture filter

port 53

Display filter

dns.qry.name.len > 50

Hva se etter? Lange eller mange DNS-forespørsler til samme domene, TXT-records:

dns.qry.type == 16

3. Tolkning av funn

  • ARP-spoofing: Hvis du ser flere ARP-replies som binder samme IP til forskjellige MAC-adresser, er det et sterkt tegn på spoofing.
  • DNS-tunneling: Uvanlig lange DNS-navn, hyppige forespørsler med varierende subdomener, eller mange TXT-records kan indikere tunneling.

Ekstra tips

  • Bruk Coloring Rules for å markere ARP og DNS-pakker med spesifikke kriterier.
  • Sett opp ring buffer for langtidsopptak.
  • Bruk Expert Information for å finne protokollfeil og advarsler.
